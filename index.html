第三方命令和一些工具集

前言

此工具集意在方便操作 windows cmd 控制台，你可以根据你的需求更改

基于 CMD 的第三方工具库，从而更方便的搭建自己的学习平台，从 c/c++ 调用系统 api 到批处理作为媒介混合各类工具的联合调度。

他的原型是为了方便 gcc/g++ 编译，所以也就解释了文件夹的名称，他使用的是FILE 目录下的 gcc 3.4.0 特别版 ，本着轻量化的原则，后续也收集了一些体积较小的工具，由于为了兼容，大部分命令为x86，即32位程序

BAT [打印+语法高亮 ]

  • 介绍：默认就带自动翻页、行号、语法高亮、Git 集成等功能的升级版文件查看工具。[ bat.exe + autocomplete\ ] [5.65+0.02MB]

  • 地址：https://github.com/sharkdp/bat [ windows中文编码支持需要调整为UTF-8 ]

结语

通过 Helps.bat 的展示，你可以看见很多在命令行实现的效果，当初黑漆漆的 cmd 窗口竟然还能带给我们这样的惊喜

目录结构

先看目录结构 [ tree -N -d compile ]

  • 宏命令在 Termux.bat 内定义
  • Compile-bin 目录下是外部命令
  • Sourse Code 目录下是部分外部命令源码
  • Sourse Com 是启用 com -c 命令后才起作用
  • x86 目录本意是存放 32 位外部命令
  • Compile-include 目录下收集了体积较小的工具
  • Home 目录是测试目录
  • FILE 目录下收集了体积较大工具

Termux.bat

下载

  • 下载精简版

操作方法

右键桌面或者资源管理器，点击Termux ，前提是配置了环境，没有配置双击运行入口文件 Termux.bat

附加说明：您只需安装一次 TERMUX-VCC 至目标电脑即可运行，如果版本有更新初始页面会提示升级，运行 install -d 下载更新

配置自己环境

由于为了缩减termux的开启时间，我们通过把不常用的宏命令定义在vcc.bat中,其中可以call my_path.cmd来加载我们自己配置的环境，有人就问了，我为什么不直接在vcc中定义，或者termux中，当然是可以的，但为了每次更新不被覆盖，这是目前较为好的方案。

一些提示

Q: 我该怎么启动此集合，安装完不知道怎么开启？

A: 你可以右键桌面或者资源管理器，点击Termux将在此路径下打开控制台，win11在更多选项那里，或者按住shift键右键

操作系统版本：

  • 系统版本要求: Win7/Win8/Win8.1/Win10 及以上

由于精力有限、建议在64位操作系统下运行
如果您要在32位上使用，一些功能可能无法使用

一些推荐

DUST [目录大小]

  • 介绍：能够一目了然地展示目录和文件大小的命令行工具。使用时无需加额外的参数，即可展示当前目录下的文件和目录的大小、包含的子目录列表（树状）以及占用空间的百分比（条形图）。[1.95MB]

地址：https://github.com/bootandy/dust

DUF [磁盘情况]

  • 介绍：通过彩色表格的方式展示磁盘使用情况的工具。不仅对设备进行了分类，还支持结果排序。[2.39MB]

地址：https://github.com/muesli/duf

BOTTOM [监控进程]

  • 介绍：图形化实时监控进程和系统资源的工具。支持实时展示 CPU、内存、硬盘、网络、进程、温度等指标，而且还可通过插件扩展可视化效果，相较于其它同类型的开源项目，该项目更加活跃。[ btm.exe + completion\ ] [3.33MB+0.02MB]

  • 地址：https://github.com/ClementTsang/bottom

HYPERFINE

  • 介绍：强大的命令行基准测试工具。支持任意 shell 命令、多次运行的统计分析和结果导出。
  • [ hyperfine.exe + autocomplete\ ] [1.13+0.02MB]

  • 地址：https://github.com/sharkdp/hyperfine

CLOC [代码审计]

  • 介绍：CLOC 对许多编程语言中源代码的空白行、注释行和物理行进行计数 [9.47MB]

  • 地址：https://github.com/AlDanial/cloc

ripgrep [ grep命令 ]

  • 介绍：超快的文本搜索工具。不仅快还更加人性化，比如自动忽略 .gitignore 中的文件、自动递归搜索、自动高亮展示匹配的结果。[ rg.exe complete\ ]

  • 地址：https://github.com/BurntSushi/ripgrep
